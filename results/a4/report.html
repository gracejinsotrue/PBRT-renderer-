The dielectric `sample()` method uses the uniform random sample to choose between reflection and refraction with probability equal to the Fresnel reflectance `Fr`. The code is implemented in a new class src/path_mats.cpp and added accordingly to the CmakeLists.txt. The point is to build a path tracing integrator that generalizes BSDF sampling, this is done by extending a Whitted-style direct illumination algorithm into a full path tracer that fully accounts for indirect illumination. This is an iterative BSDSF sampling path tracer and includes both reflected radiance and emitted radiance integrators. The choice for iterative approach does not align as well with the more intuitive recursive estimator, but the iterative estimator was just easiest to implement and debug, which indeed came in handy. We use the Russian Roulette strategy where "continuation probability = min(maximum component of the throughput * eta^2, 0.99)" to match the class reference renders exactly. This is applied after three bounces to better avoid killing paths that are high energy but short. The real main differenced from the Whitted integrator is that EVERY surface rather than just specular surfaces continues the path with BSDF sampling. The benefit is that we can capture indirect illumination that a pure whitted technique would miss entirely. For example in the Cornell box, there are some nice blue and red hues in the shadow of the spheres because some rays can bring those respective colors in as indirect illumination! The implementation is in src/path_mis.cpp. This path tracing method just combines both sampling strategies from our now-implemented path_ems.cpp and path_mats.cpp with our now well-known balance herusitic from MIS. The balance heuristic I took directly from my direct integrator MIS. At each diffuse bounce, the emitter sample is weighted by $w_l = p_l / (p_l + p_b)$ and the BSDF sample (when it hits an emitter) is weighted by $w_b = p_b / (p_l + p_b)$, where both pdfs are in solid angle measure. The emitter's area pdf is converted via $p_{\text{solid angle}} = p_{\text{area}} \cdot d^2 / |\cos\theta_y|$. The only difference is for the BSDF sample to continue the path in another iteration of the loop to add contributions from later bounces. How it is accomplished in my codebase is that: In direct.cpp's MIS mode, the BSDF sample 1. traces a ray, 2. checks if it hits an emitter 3. applies MIS weight, that is all. For path_mis.cpp, the same BSDF sample does all these steps BUT the loop still continues with throughput*= bsdfWeight and builds the next ray.
